# %TYPE
`%TYPE` — це спосіб оголосити змінну, щоб її тип **автоматично відповідав** типу стовпця в таблиці або іншої змінної, на момент компіляції скрипту.
## Синтаксис
```SQL
identifiertable.column name%TYPE;
```

## Приклад
```SQL
DECLARE
v_email users.email%TYPE ;
v_my_email v_email%TYPE := 'janko.hrasko@tuke.sk';
```

# %ROWTYPE
`%ROWTYPE` — це як `%TYPE`, але для **цілого рядка** таблиці, а не одного стовпця.
## Синтаксис
```SQL
DECLARE
    v_user   users%ROWTYPE;
```

## Приклад
```SQL
DECLARE
    v_user   users%ROWTYPE;
BEGIN
    SELECT * INTO v_user FROM users WHERE id = 1;
    RAISE NOTICE 'Email: %', v_user.email;
END;
```

## RECORD
`RECORD` — це змінна без фіксованої структури, яка приймає структуру того, що в неї записується. Щось схоже на динамічний тип даних але після першого присвоєння тип змінити не можна.
## Синтаксис
```SQL
DECLARE
    r RECORD;
```

## Приклад
```SQL
DO $$
DECLARE
    r RECORD;
BEGIN
    -- перше присвоєння: r отримує структуру {id, email}
    SELECT id, email INTO r FROM users WHERE id = 1;
    RAISE NOTICE 'id: %, email: %', r.id, r.email;

    -- друге присвоєння: r отримує нову структуру {id, name, price}
    SELECT id, name, price INTO r FROM products WHERE id = 1;
    RAISE NOTICE 'id: %, name: %, price: %', r.id, r.name, r.price;

    -- тепер r.email більше не існує — буде помилка компіляції
    -- RAISE NOTICE '%', r.email; -- ERROR
END;
$$
```
