Операції над математичними множинами.
Операції з множинами в PostgreSQL:
- Об'єднання - UNION, UNION ALL.
- Перетин - INTERSECT.
- Різниця - EXCEPT.

# Властивості
- Однакова кількість стовпців має бути вибрана всіма використаними операторами SELECT. Назви стовпців, що використовуються у списку, взяті з першого запиту. 
- Типи даних списку стовпців повинні бути сумісними / неявно конвертованими. 
	- Неявне перетворення типів не виконуватиметься, якщо відповідні стовпці в запитах належать до різних груп типів даних. Наприклад, якщо стовпець у запиті для першого компонента має тип даних DATE, а відповідний стовпець у запиті для другого компонента має тип даних CHAR, PostgreSQL не виконає неявне перетворення, а викличе помилку.
- Речення ORDER BY можна використовувати лише один раз в кінці запиту, що містить складені оператори SELECT. Це означає, що окремі оператори SELECT не можуть мати речення ORDER BY. 
- Операції UNION та INTERSECT є комутативними, тобто порядок запитів не важливий (він не змінює кінцевий результат). 
- З точки зору продуктивності, операція UNION ALL працює краще, ніж UNION, оскільки вона не витрачає ресурси на фільтрацію дублікатів та сортування набору результатів. 
- Операції з множинами можуть бути частиною підзапитів.
