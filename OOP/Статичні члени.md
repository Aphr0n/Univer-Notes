Статичні дані класу це дані які спільні для всіх об’єктів класу, що робить структуру складнішою. Статичні змінні та методи дозволяють мати **глобальні дані і функції** в межах класу.  Наприклад, якщо є клас `Math` зі статичним методом `sqrt()`, тобі **не треба створювати об’єкт Math**, щоб викликати `Math.sqrt(16)`.  Для створення статичних змінних або методів слід використовувати ключ. слово `static`. Статична змінна використовує статичний режим пам'яті, а статичний метод можна викликати безпосередньо з будь-яких екземплярів класу.
# Причини використання
- Статична змінна подібна до глобальної змінної
- Значення статичної змінної зберігається у статичній пам'яті, яка є спільною для всіх об'єктів класу
- Java створює лише одну копію статичної змінної, яку можна використовувати, навіть якщо клас ніколи не створюється
- Не порушує інкапсуляцію, тому що вона є спільною лише для об'єктів класу
- Корисно, коли ми хочемо створити змінну, спільну для всіх екземплярів класу
- Одним з найпоширеніших прикладів є наявність змінної, яка може підраховувати, скільки об'єктів класу було створено
# Константи
Статичні змінні здебільшого використовуються як константи з ключовим словом final в оголошенні
```Java
public class MaxUnits {
	public static final int MAX_UNITS = 25;
}
```

# Статичні методи
Клас може мати методи, які визначені як статичні (наприклад, метод main).До статичних методів можна отримати доступ без використання об'єктів. Немає потреби створювати екземпляри. Статичні методи зазвичай використовуються для групування пов'язаних бібліотечних функцій, які не залежать від членів даних свого класу, наприклад, бібліотечні функції Math
## Обмеження статичних методів
- Вони можуть викликати лише інші статичні методи
- Вони можуть отримувати доступ лише до статичних даних
- Вони не можуть посилатися на this або super жодним чином
  
# Керівні принципи використання статичних змінних
1. Ніколи не використовуйте статичні змінні, не оголосивши їх final, якщо ви точно не розумієте, чому ви їх оголошуєте static, але статичні final змінні, або константи, часто дуже доречні.
2. Існує лише кілька ситуацій, коли використання нефінальної статичної змінної (глобальної змінної) може бути доречним.
	- Одним із доречних застосувань може бути підрахунок кількості об'єктів, створених з певного класу
	- Я підозрюю, що є й кілька інших доречних застосувань
	- Завжди максимально зменшуйте використання глобальних змінних
  
# Керівні принципи використання статичних методів
1. Не оголошуйте методи статичними, якщо є вимога до методу запам'ятовувати щось від одного виклику до наступного. Немає можливості використовувати змінні екземпляра в статичних методах
2. Метод, ймовірно, також має бути автономним.  Уся інформація, необхідна методу для виконання його роботи, повинна надходити або з вхідних параметрів, або з кінцевих статичних змінних-членів (констант).
3. Метод, ймовірно, не повинен залежати від значень, що зберігаються в нефінальних статичних змінних-членах, які можуть змінюватися з часом.
	- Статичний метод має доступ лише до інших статичних членів класу, тому він не може залежати від змінних екземпляра, визначених у класі.
	- Прикладом статичного методу є метод sqrt() класу Math. Цей метод обчислює та «повертає правильно округлений додатний квадратний корінь з числа double», де значення double надається як параметр методу. Щоразу, коли метод викликається, він завершує своє завдання та не намагається зберегти жодні значення з цього виклику до наступного. Крім того, він отримує всю інформацію, необхідну для виконання своєї роботи, з вхідного параметра.
