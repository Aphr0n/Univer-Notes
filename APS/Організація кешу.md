# Ємність кешу: C
Кеш-пам'ять має обмежену ємність
Якщо вона заповнена: програма запитує дані X → повинна звільнити місце в кеші → тому дані Y відкидаються
Виникає конфлікт, якщо програмі потрібні дані Y
Як вибрати «правильний» Y для відкидання з кешу?
Методи відкидання
- Випадковий вибір
- Вибір за прапорцем активності
- [[Стратегія LRU]] (найменше використані):
Найдовші невикористані дані (блок) відкидаються з кешу
# Розмір блоку: b
Кількість блоків у кеші: B = C/b
Кількість блоків у кадрі: N
Кількість кадрів: S = B/N
