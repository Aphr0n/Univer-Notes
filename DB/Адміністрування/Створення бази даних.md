```SQL
CREATE DATABASE database_name
WITH
[OWNER = role_name]
[TEMPLATE = template]
[ENCODING = encoding]
[LC_COLLATE = collate]
[LC_CTYPE = ctype]
[TABLESPACE = tablespace_name]
[ALLOW_CONNECTIONS = true | false]
[CONNECTION LIMIT = max_concurrent_connection]
[IS_TEMPLATE = true | false )
```

Щоб виконати команду CREATE DATABASE, потрібна роль суперкористувача або спеціальний привілей CREATEDB.
Спочатку треба ввести назву нової бази даних після ключових слів CREATE DATABASE. Назва бази даних має бути унікальною.
Параметри:
1. OWNER - Призначає роль (користувача), який буде власником бази даних. Поведінка за замовчуванням - користувач, який виконав команду.
2. TEMPLATE - Шаблон для створення нової бази даних. Поведінка за замовчуванням - database template1.
3. ENCODING - Визначає кодування символів у новій базі даних.
4. LC COLLATE - Цей параметр впливає на порядок сортування рядка в запитах, що містять речення ORDER BY.
5. LC CTYPE - Класифікація символів, яку використовуватиме нова база даних. 
6. ТАБЛИЧНИЙ ПРОСТІР – табличний простір для нової бази даних.
7. ОБМЕЖЕННЯ З’ЄДНАНЬ – Вкажіть максимальну кількість одночасних підключень до нової бази даних. Значення за замовчуванням – -1, тобто необмежено. Цей параметр корисний у середовищах спільного хостингу, де можна налаштувати максимальну кількість одночасних підключень для певної бази даних.
8. ДОЗВОЛИТИ З’ЄДНАННЯ – Логічне значення. Хибність – підключитися до бази даних неможливо.
9. IS TEMPLATE – Логічне значення. Істина – базу даних може клонувати будь-який користувач з привілеєм CREATEDB. Якщо значення дорівнює хибності, клонувати її можуть лише суперкористувачі або власник бази даних.
