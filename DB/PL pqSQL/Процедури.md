Збережена процедура — це іменований блок PL/pgSQL коду, який зберігається в базі даних і може бути викликаний повторно.

Головні характеристики:
- приймає параметри (IN, INOUT)
- не повертає значення напряму (на відміну від функцій)
- може містити транзакції (COMMIT, ROLLBACK)
- зберігається на сервері БД

# Процедури vs функції
- Процедури можуть використовувати транзакції (COMMIT, ROLLBACK) — функції не можуть
- Процедура не повертає значення — RETURN без виразу лише завершує виконання
- Щоб "повернути" значення, використовують параметри з режимом INOUT

# Створення процедур
```SQL
CREATE [OR REPLACE] PROCEDURE procedure_name(parameter_list)
AS $$
DECLARE
    -- оголошення змінних
BEGIN
    -- тіло процедури
END;
$$
LANGUAGE plpgsql;
```

## Транзакційні процедури
```SQL
CREATE OR REPLACE PROCEDURE transfer(
    sender   INT,
    receiver INT,
    amount   DEC
)
LANGUAGE plpgsql
AS $$
BEGIN
    UPDATE accounts SET balance = balance - amount WHERE id = sender;
    UPDATE accounts SET balance = balance + amount WHERE id = receiver;
    COMMIT;
END;
$$;
```
# Виклик процедур
```SQL
CALL stored_procedure_name(argument_list);
```

# Видалення процедур

```SQL
drop procedure [if exists) procedure_name (argument_list)
[cascade restrict]
```
- Після блоку видалення процедури вкажіть назву процедури, яку потрібно видалити. Або ж використовуйте оператор if exists.
- Вкажіть список аргументів процедури.
- Оскільки процедури можуть бути перевантажені, PostgreSQL має знати, яку процедуру видалити, перевіряючи список аргументів.
- Якщо ім'я процедури унікальне в схемі, немає потреби вказувати список аргументів.
- Якщо об'єкти залежать від процедури, їх можна рекурсивно видалити за допомогою блоку cascade.
