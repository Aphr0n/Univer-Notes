# UNION
UNION - створює об'єднання результатів двох (або більше) SELECT-запитів в один.
Запити повинні мати однакову реляційну схему, тобто стовпці в запитах:
- Мають мати однаковий (або сумісний) тип даних.
- Мають бути однаковою кількістю.
- У результаті немає дублікатів.
  
  ## Синтаксис
```SQL
SELECT column ( s ) FROM table 1
UNION
SELECT column(s) FROM table 2
```

## Приклад
```SQL
SELECT followee
FROM follows
WHERE follower = 'user'
UNION
SELECT friend_with
FROM friends_with
WHERE friend = 'user'
```

# UNION ALL
Те ж саме що й UNION, але є деякі відмінності
- Усі записи залишаються в результатах, дублікати не видаляються. 
- Результат не сортується. 
- Швидше, ніж UNION.

## Синтаксис
```SQL
SELECT column (s) FROM table 1
UNION ALL
SELECT column (s) FROM table 2
```
## Коли використовувати
- Якщо в таблиці є ідентичні рядки, і їх потрібно зберегти.
- Якщо в результаті не може бути дублікатів (тобто немає сенсу їх видаляти).
- Якщо не має значення, чи існують дублікати, чи ні.
# Властивості
- Псевдонім може бути лише в першому операторі SELECT.
- Може бути лише один ORDER BY, і він має бути в кінці оператора.
- Якщо типи стовпців не однакові, але сумісні, виконується автоматичне перетворення.
- Якщо типи стовпців різні, їх необхідно перетворити (наприклад, за допомогою функції TO CHAR()).
- Об'єднання таблиць з різною кількістю стовпців (шляхом додавання константи до додаткового стовпця).
