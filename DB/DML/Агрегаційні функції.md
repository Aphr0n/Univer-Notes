Агрегатні функції повертають значення, яке обчислюється зі значень у стовпці. 
- Вони об'єднують колекцію значень в одне значення.
- Значення NULL ігнорується.
За допомогою агрегатних функцій можна отримати:
- кількість усіх записів,
- мінімальне/максимальне значення,
- середнє значення,
- суму.

# Властивості
- Агрегатна функція повертає один результат для всього стовпця або таблиці.
- Агрегатні функції використовуються для отримання зведених результатів.
- Вони повертають результати на основі груп рядків.
- За замовчуванням усі рядки в таблиці вважаються однією групою.
- Речення GROUP BY оператора SELECT використовується для поділу рядків на менші групи.

## COUNT
COUNT() повертає кількість рядків, що відповідають заданим критеріям. Використовується для: записів, тексту, чисел та дат.
######## Приклад
```SQL
SELECT COUNT(*) FROM table name;
SELECT COUNT(collumn_name)
FROM table_name;
SELECT
COUNT(DISTINCT collumn_name)
FROM table_name;
```

## AVG
AVG() повертає середнє значення значень у числовому стовпці. Використовується для чисел
###### Приклади
```SQL
SELECT AVG(collumn_name) FROM table_name;

SELECT 
AVG(floor((current_date - birthday)/ 365))
FROM dbuser;
```

## MIN/MAX
MIN()/MAX() - повертає найменше/найбільше значення з заданого стовпця. Використовується на: текст, числа

## Приклад
```SQL
SELECT MAX(collumn name) FROM table name;
SELECT MIN(collumn name) FROM table name;
SELECT MAX(population) FROM state;
SELECT MIN(population) FROM state;
```

# Групування записів
## HAVING
Використовується в поєднанні з GROUP BY для фільтрації записів, що представляють групи. 
- WHERE у запиті фільтрує записи перед їх групуванням
(може включати стовпці, яких немає в GROUP BY).
- HAVING фільтрує записи після їх групування.
### Приклад
```SQL
SELECT type, count ( * )
FROM post
GROUP BY type
HAVING count ( * ) > 1 ;
```

## ROLLUP
Використовується в поєднанні з GROUP BY для включення проміжних підсумків для групування стовпців справа наліво.
## Приклад
>кількість користувачів однакової статі та віку, плюс проміжний підсумок для статі, і нарешті для всіх разом

```SQL
SELECT sex, floor((current date - birthday)/365), count(*)
FROM dbuser
GROUP BY ROLLUP (sex, floor((current date - birthday)/365));
```

## CUBE
Використовується в поєднанні з GROUP BY для включення проміжних підсумків для всіх комбінацій стовпців групування.
### Приклад
>кількість користувачів однакової статі та віку,
плюс проміжний підсумок для тієї ж статі, проміжний підсумок для того ж
віку, і, нарешті, для всіх разом

```SQL
SELECT sex,
floor((current date - birthday)/365), count(*)
FROM dbuser
GROUP BY CUBE (sex, floor((current date - birthday)/365));
```
