У PostgreSQL схема — це простір імен, що містить іменовані об’єкти бази даних, такі як таблиці, представлення, індекси, типи даних, функції, збережені процедури та оператори.
Щоб отримати доступ до об’єкта в схемі, потрібно звернутися до нього за допомогою такого синтаксису: schema_name.object_name 
База даних може містити одну або декілька схем, і кожна схема належить лише одній базі даних. 
Дві схеми можуть мати різні об’єкти з однаковим ім’ям.
Існує кілька сценаріїв, коли схеми корисні:
- Схеми дозволяють організовувати об'єкти бази даних, такі як таблиці, у логічні групи, щоб зробити їх більш керованими.
- Схеми дозволяють кільком користувачам використовувати одну базу даних, не заважаючи один одному.
PostgreSQL автоматично створює схему з назвою public для кожної нової бази даних.
Будь-який об'єкт, створений без вказівки назви схеми,
PostgreSQL розміщує в цій публічній схемі.

# Привілегії
- Користувачі можуть отримувати доступ лише до об'єктів у схемах, якими вони володіють. 
- Щоб надати користувачам доступ до об'єктів у схемі, якою вони не володіють, користувачам необхідно надати право USAGE для цієї схеми. GRANT USAGE ON SCHEMA назва_схеми TO назва_ролі; 
- Щоб дозволити користувачам створювати об'єкти у схемі, якою вони не володіють, користувачам необхідно надати право CREATE для цієї схеми. GRANT CREATE ON SCHEMA назва_схеми TO назва_ролі; 
- За замовчуванням кожен користувач у публічній схемі має права CREATE та USAGE.
# Редагування схеми
Перейменування схеми (можливо лише якщо користувач є власником схеми та має право CREATE для бази даних):
```sql
ALTER SCHEMA schema_name
RENAME TO new_name;
Зміна власника схеми:
ALTER SCHEMA schema_name
OWNER TO
{ new_owner | CURRENT_USER | SESSION_USER};
```

# Видалення схеми
Команда DROP SCHEMA видаляє схему та всі її об'єкти з бази даних:
```
DROP SCHEMA [IF EXISTS] назва_схеми
[ CASCADE | RESTRICT ];
```
>Команду DROP SCHEMA може виконати лише власник схеми або суперкористувач.

PostgreSQL дозволяє видаляти кілька схем одночасно за допомогою однієї команди 
```sql
DROP SCHEMA:
DROP SCHEMA [IF EXISTS]
назва_схеми1 [,назва_схеми2,...]
[CASCADE | RESTRICT ];
```
