**Прив’язка (binding)** — це **процес встановлення відповідності між викликом методу та конкретною реалізацією (кодом методу), яка буде виконана**.

# Рання прив’язка
Відбувається **під час компіляції**
Компілятор **заздалегідь знає**, який метод викликати
Не використовується поліморфізм
- **Приклад:**
```Java
Dog d = new Dog(); d.bark();
```
Тип змінної `d` і об’єкта відомий → викликається `Dog.bark()`.

# ### Пізня прив’язка (late / dynamic binding)
Відбувається **під час виконання програми (run-time)**
Використовується з **поліморфними посиланнями**
Рішення про те, який метод викликати, залежить від **реального типу об’єкта**, а не типу змінної
- **Приклад:**
```Java
Animal a; a = new Dog(); a.makeSound();   // викликається Dog.makeSound()
a = new Cat(); a.makeSound();   // викликається Cat.makeSound()
```
Хоча тип змінної `a` — `Animal`, фактичний метод визначається **типом об’єкта** (`Dog` або `Cat`).
