Інкапсуляція - принцип який говорить що потрібно розділяти поведінку об'єктів від її реалізації. За ним класи повинні бути непрозорими і не розкривати деталі своєї реалізації

# Приховування практичної інформації
Загальний принцип приховування інформації говорить що слід використовувати приватні елементи та відповідні їм публічні методи доступу та мутатори класу слід всюди де можливо.

# Методи доступу та мутатори класу
**Accessor (getter)** — метод, який **читає** значення класу
```Java
public double getPrice() {
    return price;
}
```

**Mutator (setter)** — метод, який **змінює** значення поля
```Java
public void setPrice(double newPrice) {
    this.price = newPrice;
}
```
## Модифікатори доступу
Доступність кожного елемента класу може бути визначеним модифікаторами доступу
- **public** 
	- доступний ззовні
	- інші об'єкти мають повний доступ до цього елемента
- **private** 
	- Доступний тільки для внутрішніх елементів класу
- Доступні для дочірніх класів

## Статичні елементи класу
Статичні елементи класу - елемент класу який належить саме класу а не об'єкту. До таких елементів автоматично мають доступ всі об'єкти класу.  Це як глобальна змінна/функція але в рамках одного класу. Є тільки 1 копія такого елементу для всього класу і для доступу до них необов'язково створювати об'єкт.
