# Продуктивність багатотактового процесора
Кількість тактів для виконання інструкції залежить від типу інструкції:
- 3 такти: beq, j
- 4 такти: інструкції R, sw, addi
- 5 тактів: lw
CPI розраховується як середньозважене значення
Бенчмарк SPECINT2000:
- 25% читання пам'яті
- 10% запису пам'яті
- 11% розгалужень
- 2% безумовних переходів
- 52% інструкцій типу R
CPI = (0,11 + 0,2)(3) + (0,52 + 0,10)(4) + (0,25)(5) = 4,12
## Найдовший шлях
$T_c = t_{pcq} + t_{mux} + max(t_{ALU} + t_{mux}, t_{mem}) + t_{setup}$
### Приклад

$T_c = t-{pcq_PC} + t_{mux} + max(t_{ALU} + t_{mux}, t_{mem}) + t_{setup} = t_{pcq_PC} + t_{mux} + t_{mem} + t_{setup} = [30 + 25 + 250 + 20] ps = 325 ps$
# Однотактовий vs багатотактовий MIPS
Нехай задана програма має 1011 інструкцій. Програма має бути виконана на багатотактному MIPS-процесорі з параметрами
- CPI = 4.12
- Tc = 325 пс
Час відгуку = (кількість інструкцій) × CPI × $T_c$= (100 × 109)(4.12)(325 × 10-12) = 133.9 секунди
Виконання триває довше, ніж у випадку однотактового процесора (92,5 секунди), через те що різні інструкції мають різний час виконання і потребу в додатковому часі обробки (tpcq + tsetup = 50 пс)
