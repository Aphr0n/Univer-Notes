# IP-адресація та мережний зв'язок (основи)

Структура адреси ІРv4 - це крапково-десяткове представлення у вигляді чотирьох десяткових чисел в діапазоні від 0 до 255. Адреси ІРу4 призначаються окремим пристроям, під'єднаним до мережі.
Примітка: Термін IP в цьому курсі використовується, коли мова йде про обидва протоколи: IPv4 та ІРvб. ІРvб - це найновіша версія IP, яка замінює поширений зараз ІРМА.
Маска підмережі IPv4 - це 32-бітове значення, яке відокремлює мережну частину адреси від вузлової частини. У поєднанні з ІРv4-адресою маска підмережі визначає, до якої підмережі належить пристрій.
Адреса шлюзу за замовчуванням - це ІР-адреса маршрутизатора, яку вузол використовуватиме для доступу до віддалених мереж, включаючи інтернет.
Адреси IPv6 мають довжину 128 бітів і записуються у вигляді рядка шістнадцяткових значень. Кожні чотири біти представлені однією шістнадцятковою цифрою; загалом 32 шістнадцяткові цифри. Групи з чотирьох шістнадцяткових цифр розділені двокрапкою ".". ІРvб-адреси не чутливі до регістру і можуть бути записані як в нижньому регістрі, так і у верхньому.











Синхронізація повідомлень також є дуже важливою в мережному зв'язку. Синхронізація повідомлення включає:
- Управління потоком (Flow Control) - процес управління швидкістю передачі даних. Контроль потоку визначає, скільки інформації може бути надіслано та швидкість, з якою вона може бути доставлена. Наприклад, якщо одна людина розмовляє занадто швидко, людині-отримувачу може бути важко почути і зрозуміти повідомлення. У мережному зв'язку є мережеві протоколи, які використовуються пристроями- відправниками та пристроями-отримувачами повідомлень для узгодження та управління потоком інформації.
- Час очікування відповіді (Response Timeout) - Якщо людина ставить запитання і не отримує відповіді протягом прийнятного проміжку часу, вона припускає, що відповіді не надходить і відповідно реагує. Людина може повторити запитання або замість нього може продовжити розмову. Вузли в мережі використовують мережеві протоколи, які вказують, як довго чекати відповіді та які дії потрібно вжити, якщо час очікування відповіді вичерпується.
- Метод доступу (Access method) - визначає, коли хтось зможе відправити повідомлення. Натисніть кнопку Відтворити на рисунку для того, щоб побачити анімацію одночасної розмови двох людей, у процесі якої відбувається «зіткнення інформації», і тому потрібно, щоб вони зупинилися і почали розмову знову. Так само, коли пристрій хоче передавати по бездротовій локальній мережі, необхідно, щоб бездротова мережна карта (WLAN МІС) визначила, чи доступне бездротове середовище.





Мережний зв'язок
Мережне передавання даних має аналогічні варіанти доставки. Як показано на рисунку, існує три типи передачі даних:
¢ Одноадресна розсилка (Unicast) - інформація передається на один кінцевий пристрій.  Багатоадресна розсилка (Multicast) - інформація передається на один або кілька кінцевих пристроїв. e Широкомовна розсилка (Broadcast) - інформація передається на всі кінцеві пристрої.







| Тип протоколу               | Опис                                                                                                                                                                                                                                                                                                                                                                                                                 |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Протоколи передавання даних | Протоколи дозволяють двом або більше пристроям спілкуватися через один або більше мереж. Сімейство технологій Ethernet передбачає використання багатьох протоколів, зокрема, протоколу IP, протокол керування передаванням (TCP), протоколу передавання гіпертекстових повідомлень (HTTP) та багатьох інших.                                                                                                         |
| Протоколи мережної безпеки  | Протоколи захищають дані забезпечуючи автентифікацію, цілісність та шифрування даних. Приклади захищених протоколів включають протокол безпечної оболонки (SSH), протокол рівня захищених сокетів (SSL) та протокол безпеки транспортного рівня (TLS).                                                                                                                                                               |
| Протоколи маршрутизації     | Протоколи дозволяють маршрутизаторам обмінюватися інформацією про маршрут, порівнювати маршрутну інформацію, а потім вибрати найкращий шлях до мережі призначення. Приклади протоколів маршрутизації включають протокол пошуку першого найкоротшого шляху (OSPF, Open Shortest Path First) та протокол граничного шлюзу (BGP, Border Gateway Protocol).                                                              |
| Протоколи виявлення служб   | Протоколи використовуються для автоматичного виявлення пристроїв або служб. Приклади протоколів виявлення послуг включають протокол динамічного налаштування вузла (DHCP, Dynamic Host Configuration Protocol), який виявляє послуги для призначення IP-адреси та систему доменних імен (DNS, Domain Name System), яка використовується для виконання встановлення відповідностей між назвами та IP-адресами вузлів. |


| Функція | Опис |
|---|---|
| **Адресація** | Ця функція ідентифікує відправника та отримувача повідомлення з використанням визначеної схеми адресації. Приклади протоколів, які забезпечують адресацію: Ethernet, IPv4 та IPv6. |
| **Надійність** | Ця функція забезпечує механізми гарантованої доставки повідомлення, у випадках якщо повідомлення втрачаються або пошкоджуються під час транспортування. TCP забезпечує гарантовану доставку. |
| **Керування потоком** | Ця функція забезпечує якомога ефективніше транспортування потоку даних між двома комунікаційними пристроями. TCP надає служби керування потоком. |
| **Послідовність** | Ця функція однозначно позначає кожен переданий сегмент даних. Пристрій-отримувач використовує інформацію про послідовність для правильного збирання інформації. Це корисно, якщо сегменти даних втрачені, отримані з затримкою або не в тому порядку. TCP надає служби послідовності. |
| **Виявлення помилок** | Ця функція використовується для визначення, чи були пошкоджені дані під час передачі. До протоколів, що забезпечують виявлення помилок, належать Ethernet, IPv4, IPv6 і TCP. |
| **Програмний інтерфейс** | Ця функція містить інформацію, яка використовується для міжпроцесної взаємодії між мережними застосунками. Наприклад, при зверненні до веб-сторінки, протоколи HTTP або HTTPS використовуються для зв'язку між процесами веб-клієнта та веб-сервера. |
