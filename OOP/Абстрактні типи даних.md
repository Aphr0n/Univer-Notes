Абстрактний клас — це **шаблон** або **каркас класу**, який описує його ідею або загальні риси, але **не можна створити його об’єкт напряму**. Абстрактний клас існує лише для того, щоб гарантувати, що його підкласи повинні перевизначити його абстрактні методи
Абстрактні методи - методи які лише оголошуються(без тіла методу)

# Конструктори абстрактного класу
- Не оголошується публічним, тому що конструктори з public призначені для типів, екземпляри яких можна створювати. Абстрактні типи ніколи не можуть бути створені.
- Оголошуються protected, для того щоб базовий клас міг виконувати завдання ініціалізації, коли створюються екземпляри похідного класу
