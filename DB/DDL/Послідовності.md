Послідовність – об’єкт бази даних, що генерує послідовність чисел $(1, 2, 3, \dots)$ . Початкове значення та крок можна встановити. Послідовність повертає наступне значення у визначеній послідовності на запит.
Використання залежить від конкретного застосування:
- Генерація ідентифікаторів у випадку штучного первинного ключа.
- Розрізнення порядку вставлених рядків.

# Послідовності в Postgresql
- Команда `CREATE SEQUENCE` створить новий генератор порядкових номерів
- Це передбачає створення та ініціалізацію нової спеціальної однорядкової таблиці з назвою.
- Послідовність належатиме користувачеві, який виконав команду.
- Назва послідовності має відрізнятися від назви будь-якої іншої послідовності, таблиці, індексу або представлення в тій самій схемі.
- Після створення послідовності можна використовувати функції nextval, currval та setval для роботи з послідовністю.
- Послідовності базуються на арифметиці bigint, тому діапазон не може перевищувати діапазон восьмибайтового цілого числа (від -9223372036854775808 до 9223372036854775807).

# Синтаксис
```SQL
CREATE SEQUENCE [ I F NOT EXISTS ] sequence_name
[ AS { SMALLINT | INT | BIGINT } ] -- тип даних. Визначає межі значень
[ INCREMENT [ BY ] increment ]
[ MINVALUE minvalue | NO MINVALUE ] -- мінімальне значення. `NO MINVALUE` → береться мінімум типу.
[ MAXVALUE maxvalue | NO MAXVALUE ] -- максимальне значення. `NO MAXVALUE` → береться максимум типу.
[ START [ WITH ] start ] 
[ CACHE cache ] -- скільки значень попередньо генерувати і зберігати в пам'яті для швидкості. За замовчуванням `1`.
[ [ NO ] CYCLE ]
[ OWNED BY { table_name.column name | NONE } ] -- прив'язати послідовність до конкретної колонки таблиці. Якщо колонку або таблицю видалити — послідовність видалиться автоматично. `NONE` знімає прив'язку.
```

## Видалення послідовності
- команда `DROP SEQUENCE`

# Приклади
## Створення послідовності
```SQL
CREATE SEQUENCE sekvencia
INCREMENT 5
START 10;
```

## Отримання актуального значення послідовності
```SQL
select currval ('sekvencia ' );
```

## Отримання насутпного значення послідовності
```SQL
select nextval ('sekvencia ' );
```

## Встановлення актуального значення послідовності
```SQL
select setval('sekvencia', 100);
```

## Створення таблицю з призначенням послідовності до колонки
```SQL
create table test (
id int primary key, sekvencia int);

alter sequence sekvencia owned by test. sekvencia;
```

## Вкладення значень і випис таблиці
```SQL
insert into test values (1, nextval ( 'sekvencia' ));
insert into test values (2, nextval ( 'sekvencia' ));
insert into test values (3, nextval( 'sekvencia' ));
select * from test;
```
