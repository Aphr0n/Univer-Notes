CREATE - створює таблицю
```SQL
CREATE TABLE DBSUser (
	username VARCHAR( 3 0 ) ,
	name VARCHAR( 3 0 ) ,
	surname VARCHAR( 4 0 ) ,
	birthday DATE,
	sex CHAR
);
```

# Рекомендований послідовність кроків створення таблиць
- Визначити типи даних окремих атрибутів (стовпців),
- визначити стовпці, які повинні та не повинні бути null,
- визначити стовпці, які повинні бути унікальними,
- визначити пари первинних та зовнішніх ключів,
- визначити значення за замовчуванням,
- визначити обмеження стовпців (на основі використаного домену),
- не використовувати зарезервовані слова SQL як імена атрибутів,таблиць тощо,
- створити таблицю.

# Обмеження атрибутів таблиць
Забезпечення межі таблиці під час вставки, зміни або видалення:
- вимога значення – NOT NULL,
	- не можна вставити значення NULL
	- значення можуть повторюватися
- унікальні значення – UNIQUE,
	- можна вставити значення NULL
	- значення не можуть повторюватися.
- первинні ключі – PRIMARY KEY,
	- не можна вставити значення NULL
	- значення не можуть повторюватися.
- зовнішні ключі – FOREIGN KEY,
- перевірка – CHECK,
- значення за замовчуванням – DEFAULT.
```SQL
CREATE TABLE o s o b a (
	username VARCHAR( 3 0 ) PRIMARY KEY,
	name VARCHAR( 3 0 ) NOT NULL,
	surname VARCHAR( 4 0 ) NOT NULL,
	birthday DATE,
	sex CHAR( 1 ) DEFAULT ’M’
) ;
```

```SQL
CREATE TABLE predmet_zapis(

```
>Первинні ключі - складений ключ (кілька стовпців)

# Зовнішні ключі
Зовнішній ключ представляє зв'язок (посилання) між стовпцями двох таблиць:
- таблиця із зовнішнім ключем: дочірня таблиця,
- таблиця, що містить первинний ключ: таблиця, на яку посилаються (батьківська).
```SQL
CREATE TABLE p r e d m e t z a p i s (
	id_predmet INT, REFERENCES predemet(id),
	id_student INT, REFERENCES student(id),
	PRIMARY KEY (id_predmetm id_student)
) ;
```
