### BNF (Backus-Naur Form)
**Синтаксис BNF:**
- `<нетермінал>` — нетермінальні символи в кутових дужках
- `::=` — замість стрілки →
- `|` — альтернатива (або)
- термінали пишуться як є: `+`, `id`, `123`
**Приклад** — арифметичні вирази:
```
<E> ::= <E> + <T> | <T>
<T> ::= <T> * <F> | <F>
<F> ::= id | (<E>)
```
**Проблема BNF:** символи `<`, `>`, `|`, `::=` зарезервовані самою нотацією — їх не можна використовувати як термінали мови.

### EBNF (Extended BNF)

Розширює BNF трьома речами:

| Нотація | Значення                        |
| ------- | ------------------------------- |
| `[...]` | необов'язково (0 або 1 раз)     |
| `{...}` | повторення (0 або більше разів) |
| `(...)` | групування                      |

Плюс — **усі термінали беруться в лапки** (`"+"`, `"0"`), тому кутові дужки для нетерміналів вже не потрібні.

**Той самий приклад в EBNF:**

```
E ::= T {"+" T}
T ::= F {"*" F}
F ::= id | "(" E ")"
```

### Як розгортати EBNF → BNF

Це важливо, бо при реалізації інтерпретатора потрібен чистий BNF.

**1. Необов'язковість `[α]`** — вводимо новий нетермінал або додаємо порожню альтернативу:

```
A ::= [α] β    →    <A> ::= α β | β
```

**2. Повторення `{α}`** — вводимо рекурсивний нетермінал:

```
A ::= {α}    →    <A> ::= <X>
                  <X> ::= <X> α | ε
```

**3. Групування `(α)`** — вводимо новий нетермінал:

```
A ::= (α | β) γ    →    <A> ::= <X> γ
                         <X> ::= α | β
```

---

Якщо коротко: **BNF — основа, EBNF — зручніший запис того самого**, просто з регулярними конструкціями для скорочення правил.
