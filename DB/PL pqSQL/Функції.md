Функції PL/SQL — це блоки коду, які можна використовувати повторно, для виконання певних завдань. Вони схожі на процедури, але завжди повинні повертати значення.

Функція в PL/SQL містить:
- Заголовок функції: Заголовок функції містить назву функції та необов'язковий список параметрів. Це перша частина функції, яка визначає назву та параметри.
- Тіло функції: Тіло функції містить виконувані оператори, що реалізують певну логіку. Воно може включати декларативні оператори, виконувані оператори та оператори обробки винятків.


# Функція vs процедура
- Функція повертає результат, тоді як збережена процедура — ні.
- Виконує певну дію, завершує дію та повертає керування викликаючій функції.
- До PostgreSQL 11 збережені процедури були реалізовані як функції, що не повертають дані.
- У поточній версії можна оголошувати збережені процедури безпосередньо.

# Декларація функції
```SQL
CREATE OR REPLACE FUNCTION factorial(x NUMBER)
  RETURN NUMBER
IS
  f NUMBER;
BEGIN
  IF x = 0 THEN
    f := 1;
  ELSE
    f := x * factorial(x - 1);
  END IF;
  RETURN f;
END;
```

# Видалення функції
```SQL
drop function [if exists] function_name (argument_list) 
[cascade restrict]
```
Після блоку видалення функції вкажіть назву функції, яку потрібно видалити. Або використовуйте if exists.
Вкажіть список аргументів для функції.
Оскільки функції можуть бути перевантажені, PostgreSQL потрібно знати, яку функцію видалити, перевіряючи список аргументів.
Якщо функція унікальна в схемі, немає потреби вказувати список аргументів.
Якщо об'єкти залежать від функції, їх можна рекурсивно видалити за допомогою блоку cascade.
# Режими параметрів функцій у PL/pgSQL
- IN — вхідний параметр (параметр в дужках)
	- Значення передається у функцію
	- Поводиться як константа — не можна змінити всередині функції
	- Якщо не вказати режим явно, параметр буде IN

```SQL
CREATE OR REPLACE FUNCTION infunkcia(x int) RETURNS int AS $$
BEGIN
    x := x + 1;  -- ПОМИЛКА: x це константа, не можна змінити
    RETURN x;
END;
$$ LANGUAGE plpgsql;
```

- OUT — вихідний параметр
	- Використовується для повернення значень з функції
	- Поводиться як неініціалізована змінна — потрібно їй присвоїти значення
	- Корисно коли треба повернути кілька значень одразу (замість одного RETURN)

```SQL
CREATE OR REPLACE FUNCTION outfunkcia(OUT x text, OUT y text) AS $$
BEGIN
    x := 1;
    y := 2;
END;
$$ LANGUAGE plpgsql;
```
- INOUT — вхідно-вихідний параметр
	- Комбінація обох: приймає значення і повертає змінене
	- Поводиться як звичайна змінна — можна читати та змінювати
```SQL
CREATE OR REPLACE FUNCTION swap(INOUT x int, INOUT y int)
LANGUAGE plpgsql AS $$
BEGIN
    SELECT x, y INTO y, x;  -- міняємо місцями
END;
$$;
```

# Функції що повертають таблицю
Замість одного значення, функція може повернути цілу таблицю — це робиться через RETURNS TABLE(...)
```SQL
CREATE OR REPLACE FUNCTION vratmesto(vzor VARCHAR)
RETURNS TABLE ( 
    mesto_meno       VARCHAR,
    mesto_populacia  INT
)
AS $$
BEGIN
    RETURN QUERY
        SELECT nazov, populacia
        FROM mesto
        WHERE nazov ILIKE vzor;
END;
$$ LANGUAGE plpgsql;
```

- визначаємо структуру таблиці що повернеться: назви колонок і їх типи 
- замість RETURN x виконуємо SELECT і його результат стає результатом функції
- Колонки SELECT-у мають відповідати колонкам з `RETURNS TABLE` за порядком і типами
- ми взяли дані з users і всунули їх в псевдо-таблицю оголошену в функції

# Перевантаження функції
PostgreSQL дозволяє кільком функціям використовувати однакове ім'я, якщо вони мають різні аргументи. 
Якщо дві або більше функцій мають однакове ім'я, імена функцій перевантажуються. 
Під час виклику перевантаженої функції PostgreSQL вибирає найкращу функцію-кандидат для виконання на основі списку аргументів функції.
```SQL
CREATE OR REPLACE FUNCTION pretazenie ()
	returns int AS $$
	DECLARE
		x int;
	BEGIN
		x := 1;
	RETURN x;
END; $$ LANGUAGE plpgsql;
CREATE OR REPLACE FUNCTION pretazenie (y int)
	returns int AS $$
	DECLARE
		x int;
	BEGIN
		x := y + 1;
	RETURN x;
END; $$ LANGUAGE plpgsql;
select pretazenie ();
select pretazenie (2);
```
