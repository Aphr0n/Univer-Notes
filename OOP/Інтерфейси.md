Інтерфейс - набір методів котрі кожен об'єкт має реалізувати. На відміну від інтерфейси містять лише абстрактні методи і константи.
```Java
public interface InterfaceName {

}
```

# Розширення інтерфейсів
 1. По суті, інтерфейс – це підмножина всіх методів, які реалізує об'єкт, тому 
	- Об'єкт може мати декілька інтерфейсів
	- Можна наслідувати лише 1 клас, але реалізовувати 
```Java
public class A extends B implements I1, I2, I3 {
...
}
```
2. Інтерфейс може успадковуватися від іншого інтерфейсу
```Java
public interface A {
	void f();
}

public interface B extends A {

	void g();
	
}
public class C implements B {

	@Override
	public void f() { ... }

	@Override
	public void g() { ... }

}
```
Клас C повинен реалізувати метод g() з інтерфейсу B, а також метод f() з успадкованого інтерфейсу A

# Інтерфейси як типи
Тип даних це по суті специфічний інтерфейс об'єкту. 
- Різні об'єкти можуть мати один і той самий тип, а один і той самий об'єкт може мати багато різних типів
- Об'єкт відомий іншим об'єктам лише через його інтерфейс
- Інтерфейс — це реалізація підтипів в об'єктно-орієнтованій мові, яка описує, коли один об'єкт можна використовувати замість іншого об'єкта.
  
# Конвенція про інтерфейси
- Оскільки інтерфейс розроблений лише для додавання певної поведінки або функцій до класів, зазвичай він містить лише один або два загальні методи. Причина цього полягає в тому, що interface не є суперкласом, тому він не визначає, хто може використовувати його методи. Загалом, його метод може використовувати кожен.
```Java
public interface Runnable {
	void run();
}
```
- Згідно з домовленістю коду Java, назва інтерфейсу зазвичай є прикметником, оскільки прикметник додає певного значення іменнику.
	- `Runnable, Comparable, Clonable, Accessible`
- Імена інтерфейсів для слухача, керованого подіями, зазвичай закінчуються на Listener.
	- `ActionListener, MouseMotionListener, KeyListener`
- Деякі програмісти використовують префікс «I» для назв інтерфейсів (угорська нотація)
	- ICommand, IMessage
