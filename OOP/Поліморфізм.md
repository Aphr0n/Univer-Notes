**Поліморфізм** - це властивість об’єктно-орієнтованих мов, за якої **один і той самий інтерфейс (тип)** може мати **різну поведінку залежно від реального класу об’єкта**.

# Поліморфізм підтипів
Об’єкти підкласів можна використовувати як об’єкти батьківського класу
- Дозволяє писати функцію так, щоб вона приймала об’єкт певного типу T, але також правильно працювала, якщо їй передано об’єкт типу S, який є підтипом T.
- **Підтипи, наслідування**
```Java
Article a = new DiscountArticle();
```

# Поліморфізм поведінки
- Поліморфні функції, які можна застосовувати до аргументів різних типів, але які поводяться по-різному залежно від типу аргументу, до якого вони застосовуються.
- **Перевантаження функцій** (наприклад, перевантаження конструкторів та методів).

```Java
class Article {
    double getPrice() {
        return price;
    }
}

class DiscountArticle extends Article {
    @Override
    double getPrice() {
        return price * (1 - discount);
    }
}

```

# Параметричний поліморфізм
- Дозволяє писати функцію або тип даних узагальнено, так щоб вони могли обробляти значення однаково, не залежачи від їхнього типу.
- **Узагальнене програмування (generic programming)**.
