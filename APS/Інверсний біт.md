Інверсний код — це знакове подання чисел, де:
додатні числа → звичайний двійковий запис
від’ємні числа → побітова інверсія додатного числа
Формула зі слайдів:
$N_IB = N$ якщо $N ≥ 0$
$N_IB = 2^n − 1 − N$  якщо $N < 0$
Як формується від’ємне число:

Беремо додатне число
Інвертуємо ВСІ біти
Готово (нічого додавати не треба)
Приклад (4 біти)
+6
```
0110
```
−6 (інверсія всіх бітів)
```
1001
```
Діапазон значень
Для N біт:
```
[ −(2^(N−1) − 1) , +(2^(N−1) − 1) ]
```
Для 4 біт:
```
[ −7 , +7 ]
```
Недоліки інверсного коду
❌ Дві нулі:
```
0000  (+0)
1111  (−0)
```
❌ Складніше додавання
потрібне “перенесення по колу” (end-around carry)
❌ Майже не використовується сьогодні
