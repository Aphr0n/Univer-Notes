Тип даних визначає набір значень, які може приймати стовпець (атрибутний домен).
Він дозволяє визначити цілісність домену. Категорії деяких вбудованих типів даних для роботи з:
- рядками символів (бітами),
- числовими значеннями,
- датою та часом,
- мультимедійними даними (BYTEA).

# Основні типи даних
- INTEGER – цілі числа,
- FLOAT – десяткові числа,
- NUMERIC(n,s) – число із заявленою точністю, де n - к-сть символів цілої частини, а s - к-сть символів десяткової частини
- CHAR(n) – рядок фіксованої довжини,
- VARCHAR(n) – рядок змінної довжини,
- DATE – рік, місяць, дата, час, хвилини, секунди,
- TIMESTAMP – розширює тип DATE частками секунд.

# Псевдо тип SERIAL
- PostgreSQL має спеціальний тип генератора об'єктів бази даних під назвою SERIAL.
- Він використовується для генерації послідовності цілих чисел, які часто використовуються як первинний ключ таблиці.
	`Syntax: collumn_name SERIAL`
- Приклад:
```SQL
CREATE TABLE tabulka1 (
	id SERIAL PRIMARY KEY,
	name VARCHAR(10)
);
```

## Використання SERIAL на стовпцях
- Початковий крок генерує послідовності цілих чисел, де наступне значення за замовчуванням залежить від значення для стовпця.
- Оскільки SERIAL завжди генерує послідовність цілих чисел, важливо встановити обмеження NOT NULL для заданого стовпця.
- Нарешті, потрібно встановити власника послідовності для стовпця ID.
- PostgreSQL підтримує 3 псевдотипи для генерації послідовностей цілих чисел, а саме `SMALLSERIAL`, `SERIAL` та `BIGSERIAL`.
