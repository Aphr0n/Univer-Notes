# Підтипи
**Підтипи (subtypes)** — це поняття , яке описує **відношення між типами**, коли **один тип можна використовувати замість іншого**. Якщо Якщо **B є підтипом A** (записують `B ≤ A`), то будь-де, де програма очікує об’єкт типу A, можна безпечно використати об’єкт типу B

```Java
Article a = new Article();          // OK
Article a = new DiscountArticle();  // OK (підтип підставляється)
DiscountArticle a = new Article();  // ❌ ПОМИЛКА
				DiscountArticle <= Article
```

# Наслідування
**Наслідування (inheritance)** — це 1 з варіантів реалізації підтипів, який дозволяє **створювати нові класи на основі вже існуючих**, автоматично **успадковуючи їхні властивості та поведінку**.

```Java
// Наслідування
class B extends A
	B is a subtype of A
	B inherits from A

//Підтиаи
class C implements D
	C is a subtype of D
```

# Переваги й ндоліки
**Переваги:**
- Нова реалізація легка, оскільки більшість коду успадковується
- Легко модифікувати або розширювати реалізацію, що повторно використовується

**Недоліки:**
- Порушує інкапсуляцію, оскільки підклас має доступ до деталей реалізації суперкласу
- “Біло-коробкове” повторне використання, оскільки внутрішні деталі суперкласів часто видимі підкласам
- Підкласи можуть потребувати змін, якщо реалізація суперкласу змінюється
- Реалізацію, успадковану від суперкласу, неможливо змінити під час виконання
