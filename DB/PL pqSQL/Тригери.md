Тригер у PostgreSQL — це функція, яка викликається щоразу, коли відбувається подія, така як вставка, оновлення, видалення або скорочення. Це спеціальна функція, пов'язана з таблицею.
Основна відмінність між функцією та тригером полягає в тому, що тригер автоматично викликається, коли виникає певна ситуація.
PostgreSQL підтримує два типи тригерів. Тригери рівня рядків та рівня операторів. Різниця між цими двома типами полягала в тому, як часто та в який час викликається тригер.

# Ствоерння тригерів

## Крок 1 — Тригерна функція
Спочатку треба написати функцію, яка містить логіку тригера:
```sql
CREATE FUNCTION trigger_function()
RETURNS TRIGGER
LANGUAGE PLPGSQL
AS $$
BEGIN
    -- логіка тут
    RETURN NEW;  -- повертаємо змінений рядок
END;
$$;
```

**Важливі обмеження:**
- не приймає параметрів
- повертає тип TRIGGER (не INT, не TABLE — саме TRIGGER)
- всередині доступні спеціальні змінні NEW і OLD

## Крок 2 — Сам тригер
```SQL
CREATE TRIGGER trigger_name
{BEFORE | AFTER | INSTEAD OF} {INSERT | UPDATE | DELETE | TRUNCATE}
ON table_name
[FOR EACH {ROW | STATEMENT}]
EXECUTE PROCEDURE trigger_function();
```

## Параметри тригерів

### Час виконання

| Параметр | Варіанти   | Що означає                      |
| -------- | ---------- | ------------------------------- |
| Час      | BEFORE     | Виконати до зміни               |
| Час      | AFTER      | Виконати після зміни            |
| Час      | INSTEAD OF | Замість зміни (тільки для VIEW) |

---

### Подія

| Параметр | Варіанти | Що означає             |
| -------- | -------- | ---------------------- |
| Подія    | INSERT   | Додавання запису       |
| Подія    | UPDATE   | Оновлення запису       |
| Подія    | DELETE   | Видалення запису       |
| Подія    | TRUNCATE | Повне очищення таблиці |
### Рівень спрацювання

| Тип                | Опис                                    |
| ------------------ | --------------------------------------- |
| FOR EACH ROW       | Спрацьовує для кожного рядка окремо     |
| FOR EACH STATEMENT | Спрацьовує один раз на всю SQL-операцію |
# NEW i OLD
Це два спеціальних об'єкти, які автоматично доступні в тілі тригера рівня рядків. Вони представляють стан рядка до і після зміни.
Уяви, що таблиця — це Excel, а рядок — це один запис. Коли щось змінюється, тригер "бачить" і старий, і новий вигляд цього рядка.

| Операція | OLD                    | NEW                |
| -------- | ---------------------- | ------------------ |
| INSERT   | ❌ NULL                 | ✅ новий рядок      |
| UPDATE   | ✅ рядок до змін        | ✅ рядок після змін |
| DELETE   | ✅ рядок що видаляється | ❌ NULL             |
## Приклад
```SQL
-- Основна таблиця
CREATE TABLE test (id int);

-- Таблиця-лог
CREATE TABLE test_log (id int, datum date);
-- Тригерна функція: при вставці в test → записати лог
CREATE FUNCTION update_test()
RETURNS TRIGGER
LANGUAGE PLPGSQL
AS $$
BEGIN
    INSERT INTO test_log(id, datum)
    VALUES (NEW.id, current_date);  -- NEW.id = id щойно вставленого рядка
    RETURN NEW;
END;
$$;
-- Тригер: викликати update_test() після кожного INSERT в test
CREATE TRIGGER test_trigger
AFTER INSERT
ON test
FOR EACH ROW
EXECUTE PROCEDURE update_test();
```

# Перемикання тригерів
## Вимкнення тригерів
```SQL
ALTER TABLE table_name
ENABLE TRIGGER trigger_name | ALL
```
## Увімкнення тригерів

```SQL
ALTER TABLE table_name
DISABLE TRIGGER trigger_name | ALL
```

# Видалення тригера
```SQL
DROP TRIGGER [IF EXISTS] trigger_name
ON table_name [ CASCADE | RESTRICT]
```
