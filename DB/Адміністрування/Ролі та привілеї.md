PostgreSQL використовує ролі для представлення облікових записів користувачів. Він не використовує концепцію користувача, як інші SRBD. Ролі, які мають права входу, називаються ролями входу. Вони еквівалентні ролі користувача в інших SRBD. 
- Коли ролі містять інші ролі, вони називаються груповими ролями - GROUPS.
```SQL
CREATE ROLE name [ [ WITH ] option [ ... ] ]
kde option môže byť:
SUPERUSER | NOSUPERUSER
| CREATEDB | NOCREATEDB
| CREATEROLE | NOCREATEROLE
| INHERIT | NOINHERIT
| LOGIN | NOLOGIN
| CONNECTION LIMIT connlimit
| [ ENCRYPTED ] PASSWORD ’password’ | PASSWORD NULL
| VALID UNTIL ’timestamp’
| IN ROLE role_name [, ...]
| ROLE role_name [, ...]
```
# Опції
1. SUPERUSER – нова роль "суперкористувач", яка може перевизначити всі обмеження доступу в базі даних. Тільки суперкористувач може створити нового суперкористувача. Якщо не вказано, значення за замовчуванням – NOSUPERUSER.
2. CREATEDB – дозволить новій ролі створювати бази даних. За замовчуванням NOCREATEDB.
3. CREATEROLE – дозволить новій ролі створювати інші ролі. За замовчуванням NOCREATEROLE.
4. INHERIT – роль успадкує привілеї будь-якої ролі, членом якої вона є. За замовчуванням INHERIT.
5. LOGIN – можливість увійти. За замовчуванням NOLOGIN.
6. CONNECTION LIMIT connlimit — якщо роль може входити в систему, визначає, скільки одночасних підключень може мати роль. -1 (за замовчуванням) означає без обмежень.
7. [ ENCRYPTED ] PASSWORD 'password' — PASSWORD NULL — встановлює пароль ролі.
8. VALID UNTIL 'timestamp' — встановлює дату і час, після яких пароль ролі більше не буде дійсним. Якщо цей параметр пропущений, пароль буде дійсний безстроково.
9. IN ROLE role_name — одна або кілька існуючих ролей, до яких нова роль буде негайно додана як новий учасник.
10. ROLE role_name — одна або кілька існуючих ролей, які будуть автоматично додані як учасники нової ролі. (Робить нову роль груповою роллю.) 

# Привілеї

## GRANT — надання привілеїв
### Синтаксис
```sql
GRANT privilege_list | ALL
  ON table_name
  TO role_name;
```
`privilege_list`: `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `TRUNCATE`…  
`ALL` — надає ролі всі наявні привілеї над таблицею.

### Приклади
```sql
-- Всі привілеї над однією таблицею
GRANT ALL ON studenti TO student;

-- Всі привілеї над всіма таблицями у схемі
GRANT ALL ON ALL TABLES IN SCHEMA public TO student;
```

## REVOKE — відкликання привілеїв

Скасовує раніше надані привілеї на об'єкти БД для вказаної ролі.

```sql
REVOKE privilege | ALL
  ON TABLE table_name | ALL TABLES IN SCHEMA schema_name
  FROM role_name;
```

## CREATE ROLE — повний приклад: користувач + група

```sql
CREATE ROLE pouzivatel WITH LOGIN PASSWORD 'SuperHeslo';
CREATE ROLE skupina;

GRANT USAGE ON SCHEMA prednasky TO skupina;
GRANT SELECT ON ALL TABLES IN SCHEMA prednasky TO skupina;

GRANT skupina TO pouzivatel;
```

> [!info] Успадкування прав
> Користувач `pouzivatel` автоматично успадкує всі права, надані групі `skupina`.

## ALTER ROLE — зміна атрибутів ролі

```sql
ALTER ROLE role_name [WITH] option;
```

Параметри: `SUPERUSER`, `NOSUPERUSER`, `CREATEDB`, `NOCREATEDB`, `CREATEROLE`, `NOCREATEROLE`, `INHERIT`, `NOINHERIT`, `LOGIN`, `NOLOGIN`, `CONNECTION LIMIT`, `PASSWORD`, `VALID UNTIL`

| Хто змінює          | Що може змінювати                         |
| ------------------- | ----------------------------------------- |
| Суперкористувач     | Будь-який атрибут для будь-якої ролі      |
| Роль з `CREATEROLE` | Будь-який атрибут, крім суперкористувачів |
| Звичайна роль       | Тільки власний пароль                     |

## DROP ROLE — видалення ролі

```sql
DROP ROLE [IF EXISTS] target_role;
```

- Видалити суперкористувача може лише інший суперкористувач
- Для видалення звичайної ролі потрібен привілей `CREATEROLE`
- Якщо на роль є посилання в БД → PostgreSQL видасть помилку

> [!warning] Перед видаленням ролі з об'єктами у БД їх потрібно прибрати або передати.

```sql
-- Варіант 1: видалити всі об'єкти ролі
DROP OWNED BY target_role;

-- Варіант 2: передати об'єкти іншій ролі
REASSIGN OWNED BY target_role TO other_role;

-- Потім:
REVOKE ALL ON ... FROM target_role;
DROP ROLE target_role;
```
