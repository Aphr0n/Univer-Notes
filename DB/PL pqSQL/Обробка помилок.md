Якщо в блоці виникає помилка, PostgreSQL перериває виконання блоку, а також відповідної транзакції. Щоб відновитися після помилки (дескриптор), потрібно використовувати пропозицію Exception у блоці ... begin ... end....

```SQL
<<label>>
begin
statements;
exception
when condition [or condition...) then
handle_exception;
[when condition [or condition... then
handle_exception;]
[when others then
handle_other_exceptions;
[
end;
```

# Команда Exception
- Якщо помилка виникає в блоці між begin та exception, PL/pgSQL зупиняє виконання та передає керування розділу exception.
- PostgreSQL переглядає умови зверху вниз, шукаючи перший збіг
- Якщо в умовах не знайдено збігу, речення exception when other перехоплює його.
- Якщо when other не вказано, виконання переривається.
- Наприклад, умова може бути такою:
	- `no_data_found`, якщо запит не повертає рядків,
	- `too_many_rows`, якщо запит повертає більше одного рядка.
# Приклад
```SQL
do
$$
declare
	rec record;
	populaciamalo int = 10000000;
begin
	select nazov, populacia
	into strict rec
from stat
where populacia populaciamalo;
/*catch exception */
exception
	when sqlstate 'P0002' then
	raise exception
	nenajdeny', populaciamalo; ako
	when sqlstate 'P0003' then
	viac ako%', populaciamalo; raise exception
end;
$$
language plpgsql;
```
