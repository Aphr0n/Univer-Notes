## Командні режими 
З міркувань безпеки Cisco IOS використовує два окремих командних режими для доступу до адміністративних функцій:
¢ Користувацький режим EXEC (User EXEC) - цей режим має обмежені можливості, але корисний для основних операцій. В користувацькому режимі доступне лише обмежене число основних команд моніторингу та неможливо виконувати будь-які команди, які можуть змінити конфігурацію самого пристрою. Користувацький режим ЕХЕС ідентифікується за допомогою запиту CLI, який закінчується символом >.
¢ Привілейований режим EXEC (Privileged EXEC) - для виконання команд конфігурації адміністратор мережі повинен отримати доступ до привілейованого режиму ЕХЕС. Режими конфігурації більш високого рівня, як, наприклад, режим глобальної конфігурації, можуть бути доступні лише з привілейованого режиму ЕХЕС. Привілейований режим ЕХЕС може бути визначений підказкою, що закінчується символом й.

Два поширених підрежими конфігурації:
- Режим конфігурації лінії - Використовується для налаштування доступу до консолі, SSH, Telnet або AUX. 
- Режим конфігурації інтерфейсу - Використовується для налаштування порту комутатора або мережного інтерфейсу маршрутизатора.

Щоб перейти з користувацького режиму EXEC в привілейований, використовуйте команду enable. Для повернення в користувацький режим EXEC використовуйте
команду привілейованого режиму disable.
Щоб перейти в режим глобальної конфігурації і вийти з нього, використовуйте команду configure terminal
привілейованого режиму EXEC. Щоб повернутися до привілейованого режиму EXEC, введіть команду exit
режиму глобальної конфігурації.

# 2.4.3 Налаштування паролів (Cisco)

## Користувацький режим EXEC

Після підключення до пристрою ви потрапляєте в режим **EXEC**. Він може бути захищений паролем.

###  Налаштування пароля для консолі

```
enable  
configure terminal  
line console 0  
password cisco  
login  
end
```

> Тепер для доступу через консоль потрібно вводити пароль.

---

## Привілейований режим EXEC

Для повного доступу до команд використовується привілейований режим.

### Встановлення enable secret

configure terminal  
enable secret class  
exit

>Команда `enable secret` забезпечує захищений доступ (з шифруванням).

---

##  Віддалений доступ (VTY: Telnet / SSH)

Лінії VTY використовуються для віддаленого підключення.

> Зазвичай доступні лінії: **0–15**

### Налаштування пароля для VTY

configure terminal  
line vty 0 15  
password cisco  
login  
end
>Це дозволяє підключення через Telnet або SSH із паролем.

show running-config, щоб перевірити, що паролі тепер зашифровані.


Щоб створити банерне повідомлення дня Ha мережному пристрої, використовуйте команду глобальної конфігурації banner motd # the message of the day #. Символ "#" у синтаксисі команд називається символом розмежування. Він вводиться до і після повідомлення


Є два системні файли, які зберігають конфігурацію пристрою:
- startup-config - Це збережений файл конфігурації, який зберігається в NVRAM. Він містить усі команди, які будуть використовуватися пристроєм під час запуску або перезавантаження. Флеш-накопичувач не втрачає свого вмісту, коли пристрій вимкнено.
- running-config - Це файл поточної конфігурації, який зберігається в оперативній пам'яті (RАМ). В ньому відображена поточна конфігурація. Зміна поточної конфігурації негайно впливає на роботу пристрою Cisco. Оперативна пам'ять (RАМ) - енергозалежна пам'ять. Вона втрачає весь свій вміст при вимкненні або перезавантаженні пристрою.
