`RAISE` — це команда для виводу повідомлень або виклику помилок у PL/pgSQL.

---
## Рівні повідомлень

| Рівень      | Поведінка                                      |
| ----------- | ---------------------------------------------- |
| `DEBUG`     | для розробника, зазвичай не видно              |
| `LOG`       | пишеться в лог сервера                         |
| `INFO`      | інформаційне повідомлення                      |
| `NOTICE`    | найчастіше використовується                    |
| `WARNING`   | попередження                                   |
| `EXCEPTION` | зупиняє транзакцію *(рівень за замовчуванням)* |

---

## Базовий синтаксис

```sql
RAISE NOTICE 'Привіт, %!', 'світ';
-- Привіт, світ!
```

```sql
RAISE WARNING 'Значення: % і %', 42, 'test';
```

> Кількість `%` має відповідати кількості аргументів, інакше буде помилка.

---

## Виклик помилки з додатковою інформацією

```sql
RAISE EXCEPTION 'duplicate email: %', email
    USING hint = 'check the email again';
```

---

## Опції `USING`

| Опція     | Що додає               |
| --------- | ---------------------- |
| `message` | текст помилки          |
| `hint`    | підказка як виправити  |
| `detail`  | детальний опис         |
| `errcode` | код помилки PostgreSQL |

---

## Якщо не вказати рівень

```sql
RAISE 'щось пішло не так';
```

Це те саме, що:

```sql
RAISE EXCEPTION 'щось пішло не так';
```

> Транзакція буде зупинена.

---

## Приклад з кількома параметрами

```sql
DO $$
DECLARE
    username TEXT := 'admin';
BEGIN
    RAISE NOTICE 'Користувач: %', username;

    RAISE WARNING 'Небезпечна дія для %', username;

    RAISE EXCEPTION 'Помилка доступу для %', username
        USING hint = 'check user permissions';
END;
$$;
```
