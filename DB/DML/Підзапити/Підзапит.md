Підзапит – запит усередині запиту (підзапит). 
- Внутрішній або вкладений запит. 
- Оператор SELECT всередині іншого оператора.
Зовнішній запит – запит, що містить підзапит. 
Не завжди необхідний. 
- Часто може бути замінений зовнішнім/внутрішнім JOIN, який зазвичай є ефективнішим (рекомендовано). 
- Велика кількість підзапитів знижує читабельність.

# Приклади використання
- У PostgreSQL підзапити можуть бути вкладеними в оператори SELECT, INSERT, UPDATE, DELETE, SET або DO, або в інший підзапит. 
- Підзапит зазвичай додається до речення WHERE іншого оператора SQL SELECT. 
- Ви можете використовувати оператори порівняння, такі як >, < або =. 
- Оператор порівняння також може бути багаторядковим оператором, таким як IN, ANY, SOME або ALL. 
- Підзапит можна розглядати як внутрішній запит, тобто SQL-запит, розміщений як частина іншого запиту, який називається зовнішнім запитом. 
- Внутрішній запит виконується першим, перед його батьківським запитом, щоб результати внутрішнього запиту можна було передати зовнішньому запиту.
# Приклад підзапиту
```SQL
select m.nazov, m.populacia, m.poznamka
from mesto m
where m.populacia <
	(select min(s.populacia) from stat s)
```

# Правила використання підзапитів
- Підзапит має бути взятий у дужки.
- Використовуйте однорядкові оператори з однорядковими підзапитами та багаторядкові оператори з багаторядковими підзапитами.
- Якщо підзапит (внутрішній запит) повертає значення null до зовнішнього запиту, зовнішній запит не повертатиме жодних рядків, якщо в реченні WHERE використовуються певні оператори порівняння.

# Порівняння за допопмого підзапитів
- Poddopyt je možné použiť pred alebo po niektorom z porovnávacích operátorov.
- Poddopyt môže vrátiť najviac jednu hodnotu.
- Hodnota môže byť výsledkom aritmetického výrazu alebo stĺpcovej funkcie.
- Potom SQL porovnáva hodnotu, ktorá je výsledkom poddopytu, s hodnotou na druhej strane operátora porovnania.
- Môžeme použiť nasledujúce operátory porovnania: =, >, >=, <, <=, ! =, <>, <=>
