Порушення цілісності посилань:
- Видалення запису, на який посилається інший запис.
- Вставка/оновлення посилання на неіснуючий запис.
Типи доступу:
- Каскадний доступ: CASCADE
		Якщо рядок у таблиці, на яку посилаються, видаляється, рядки у відповідних таблицях, на які посилаються, також видаляються.
- Обмежений доступ: RESTRICT (NO ACTION)
		Запис не можна видалити, якщо на нього посилається зовнішній ключ.
- Null-доступ: SET NULL
		Якщо відповідний запис у таблиці, на яку посилаються, видаляється або оновлюється, зовнішній ключ встановлюється на NULL.
Поведінка за замовчуванням (PostgreSQL):
```SQL
ON DELETE NO ACTION
```

## Приклад
```SQL
CREATE TABLE predmet_zapis(
id_predmet INT
id_student INT
PRIMARY KEY (id_predmet, id_student), 
FOREIGN KEY(id_predmet)
REFERENCES predmet(id) ON DELETE CASCADE,
FOREIGN KEY(id_student)
REFERENCES student(id) ON DELETE CASCADE
);
```
